JS JavaScript
Async / Await & Fetch
The game loads and saves all player data from Firebase using fetch() with async/await. This is one of the most important modern JS patterns to learn.
Why Async?

Fetching data from a server takes time — milliseconds to seconds. JavaScript can't just pause and wait (that would freeze the whole page). Async/await lets you write code that looks sequential but actually waits in the background.

Timeline — Loading a Player Save
T+0ms — Fetch sent
fetch(`${FB_URL}/saves/${myId}.json`)
T+0ms to ~200ms — Awaiting response
JavaScript continues running other code here — the page stays responsive
T+~200ms — Response arrives
const state = await res.json()
T+~205ms — State applied
restoreState(state) — UI updates
Real Firebase Load — From the Game
JS — Loading buildings (index.html ~line 6283)async function loadBuildingsFromFirebase() { const myId = getOrCreatePlayerId(); try { // Fetch world buildings AND player save — IN PARALLEL const [bRes, sRes] = await Promise.all([ fetch(`${FB_URL}/world/buildings.json?cb=` + Date.now()), fetch(`${FB_URL}/saves/${myId}.json?cb=` + Date.now()), ]); // Process world buildings if (bRes.ok) { const d = await bRes.json(); if (d && !d.error) _worldBuildings = d; } // Process player save if (sRes.ok) { const s = await sRes.json(); if (s && s.lair) _lairData = s.lair; } } catch (e) { // Network failure — fail silently or show error console.error('Load failed', e); } }
Saving to Firebase — PUT request
JS — persistState() — saving player dataasync function persistState() { const id = getOrCreatePlayerId(); const url = `${FB_URL}/saves/${id}.json`; const res = await fetch(url, { method: 'PUT', // Write (overwrite) the data headers: { 'Content-Type': 'application/json' }, body: JSON.stringify(currentState), // Convert JS object → JSON string }); if (!res.ok) throw new Error('Save failed'); return await res.json(); }

Promise.all() runs multiple async operations simultaneously. The game uses it to load world data and player data at the same time — cutting load time roughly in half compared to loading them one after another.

Pattern 1 — Array.map() to HTML

The most common pattern: transform an array of data into an array of HTML strings, then join them.

JS — Rendering level-up gain rows// Data: [{stat: 'cunning', amt: 1}, {stat: 'gnosis', amt: 1}] gainsEl.innerHTML = lu.statGains .map(g => // Transform each item into an HTML string `<div class="levelup-gain-row"> ${g.stat.charAt(0).toUpperCase() + g.stat.slice(1)} <span>+${g.amt}</span> </div>` ) .join(''); // Join array of strings into one string // Result: // <div class="levelup-gain-row">Cunning<span>+1</span></div> // <div class="levelup-gain-row">Gnosis<span>+1</span></div>
Pattern 2 — filter() then map()
JS — Showing only unlocked skillsconst available = Object.entries(_cachedWorldSkills) .filter(([k, s]) => { // Keep only skills that pass ALL checks if (!s) return false; const cat = s.category || 'utility'; if (cat === 'rite') return false; // rites go to Arcana tab const classOk = !s.classKey || s.classKey === charClass; const levelOk = (s.unlockLevel || 1) <= charLevel; return classOk && levelOk; }) .map(([k, s]) => // Then turn each into HTML `<div style="border-left:2px solid ${col}..."> <span>${s.name || k}</span> <span>${s.desc}</span> </div>` ); el.innerHTML = available.join('');
Pattern 3 — Object.entries() for Key-Value Data
JS — Rendering building cards from an object// _worldBuildings = { 'alchemical-forge': {name, desc}, 'altar': {...} } grid.innerHTML = Object.entries(_worldBuildings) .map(([key, b]) => { // Destructure key and value const lvl = playerBuildings[key] || 0; const locked = lvl === 0; return ` <div class="building-card ${locked ? 'locked' : ''}"> <span>${locked ? 'Locked' : 'Lv. ' + lvl}</span> <div>${b.name || key}</div> <p>${b.desc || ''}</p> </div>`; }) .join('');
Template Literals — The Game's Superpower
JS — Template literals (backtick strings)// Old way (messy string concatenation) '<div class="card">' + name + '</div>' // Modern way — template literal with ${expressions} `<div class="card">${name}</div>` // Multi-line template literal — used everywhere in the game `<div class="acolyte-card"> <div class="acolyte-name">${a.name}</div> <div class="acolyte-class">${a.class} · Lv.${a.level}</div> <div class="stress-bar"> <div class="stress-fill" style="width:${stressPct}%;background:${stressColor};"> </div> </div> </div>` // Conditional in template literal (ternary operator) `<div class="${locked ? 'building-card locked' : 'building-card'}">`

Template literals (backtick strings) are the most used pattern in the game. They let you embed any JavaScript expression inside ${...} — variables, function calls, ternary operators, even nested template literals.
